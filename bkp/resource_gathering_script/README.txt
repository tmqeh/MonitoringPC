AIX는 메모리 사용율이 comp/noncomp 때문에 정확하지 않아서
별도로 작성했고

일반적인 리눅스는 vmstat을 사용한다.
일반적인 윈도우는 perfmon의 카운터를 사용한다.

다만 vmstat과 perfmon 에서 추출된 데이터는 
연속적인 데이터가 아니라 배치로 간헐적으로 수집하는 경우
1번째 값이 부정확하기 때문에
수집을 할때는 반드시 1번째 값을 버리고 2번째부터 수집해서 저장할 수 있도록 한다.

target
----------------------------------------
서버 접속 스트링은 "WinSCP > 세션 > 세션 URL/코드 생성"을 참고한다.

gatehrALL* : 네이밍룰과 수집 기준으로 파일을 분리하였고 호출은 전체 대상
*_dr : SRDF와 BCV 일배치 데이터 수집
*_fs : filesystem의 사용량을 일배치 데이터 수집
*_ora_alert : alert Log 일배치 데이터 수집 (개발만 수집)
*_ora_storage : ASM과 테이블스페이스 사용량 일배치 데이터 수집
*_rsrc : 1시간 단위 리소스 사용량 수집

※ 모든서버를 대상으로 할 경우, *_rsrc 준실시간으로 변경할 방법을 검토해야됨

LOGFILE_RETENTION : 수집된 데이터를 데이터보관주기에 맞게 보관(이동) 및 삭제
